LiDAR Theory
How LiDAR Works:

LiDAR (Light Detection and Ranging) is a remote sensing method that uses light in the form of pulsed lasers to measure distances to the Earth's surface.

LiDAR operating principle
Key Components:
  • Laser: Emits pulses of light
  • Scanner: Directs laser pulses
  • Photodetector: Measures returned signals
  • GPS/IMU: Tracks position and orientation
Pulse Types:
Type Wavelength Use
NIR 1064nm Topographic
Green 532nm Bathymetric
Return Types:

A single laser pulse can have multiple returns:

  • First Return: Typically tree canopy
  • Intermediate Returns: Vegetation layers
  • Last Return: Often ground surface
Accuracy Factors:
  • Flying height
  • Scan angle
  • GPS/IMU precision
  • Atmospheric conditions
  • Target reflectivity
LiDAR Terminology
LiDAR Terminology Glossary - Common terms used in LiDAR mapping
AGL
Above Ground Level - Height measurement from ground surface rather than sea level.
Bare Earth DEM
Digital Elevation Model containing only ground points with vegetation and buildings removed.
DSM
Digital Surface Model - Represents the earth's surface including all objects (buildings, vegetation).
DTM
Digital Terrain Model - Representation of bare ground surface without objects.
GCP
Ground Control Point - Surveyed point with known coordinates used for georeferencing.
GSD
Ground Sampling Distance - Distance between pixel centers on the ground.
IMU
Inertial Measurement Unit - Device that measures aircraft orientation and movement.
Intensity
Strength of the return signal, affected by surface reflectivity.
LAS/LAZ
Industry-standard file formats for LiDAR point clouds (LAZ is compressed).
Point Cloud
Collection of data points in 3D space, each with X, Y, Z coordinates.
Point Density
Number of points per square meter, indicating data resolution.
RMSE
Root Mean Square Error - Statistical measure of positional accuracy.
SBET
Smoothed Best Estimate of Trajectory - Processed flight path data.
Abstract

My practicum with Eagle Mapping focused on aerial imagery acquisition and processing workflows for high-precision topographic mapping. The practicum involved flight planning, aerial data collection, data processing, and point cloud classification. Working with industry-standard equipment and software, I participated in the complete LiDAR workflow from acquisition to final product delivery, addressing challenges in data management, quality control, and integration with existing GIS systems while adhering to industry specifications for spatial accuracy.

LiDAR Workflow Overview
✨ Key Components ✨

The product workflow at Eagle Mapping involves a series of integrated steps resulting in high-precision topographic data. Flight planning ensures optimal coverage and point density, while acquisition captures millions of data points per second using aerial LiDAR sensors. Processing involves trajectory correction and data alignment, followed by classification to identify different surface types. Final products include digital terrain models (DTMs), digital surface models (DSMs), and high resolution imagery used in various GIS applications.

LiDAR Acquisition
Cessna 210 with RIEGL LiDAR scanner
Equipment Used:
  • Aircraft: Cessna 210
  • LiDAR Scanner: RIEGL VQ-780i
  • IMU/GNSS: Applanix POS AV 610
  • Data Recorder: RIEGL RiACQUIRE
RIEGL VQ-780i Specifications:
  • Measurement rate: up to 1 MHz
  • Accuracy: ±20mm
  • Precision: ±15mm
  • Maximum range: 2,500m
Data Collection Process:

During flight operations, the RIEGL VQ-780i scanner emits near-infrared laser pulses that reflect off ground features and return to the sensor. The system records the time difference between pulse emission and return to calculate distance. Simultaneously, the integrated IMU/GNSS system records precise aircraft position and orientation. Together, these systems allow for accurate 3D positioning of each laser return point in real-time.

LiDAR Processing
RIEGL RiPROCESS software interface
Processing Software:
  • RIEGL RiPROCESS
  • TerraSolid TerraScan
  • Global Mapper
Processing Workflow:
  1. Import raw LiDAR data and trajectory
  2. Apply system calibration parameters
  3. Filter outlier points and noise
  4. Adjust flight line alignment
  5. Absolute accuracy adjustment using GCPs
  6. Strip alignment and adjustment
  7. Generate preliminary point cloud
Quality Control Metrics:
  • Point density verification: 8-12 points/m²
  • Flight line overlap verification: >30%
  • Vertical accuracy: ±15cm RMSE
  • Point cloud completeness: >99%
Trajectory Processing:

A critical step in LiDAR processing involves accurately determining the aircraft's position and orientation throughout the flight. This is achieved by processing GNSS and IMU data using Applanix POSPac MMS software, generating a Smoothed Best Estimate of Trajectory (SBET) file. The SBET is then combined with raw laser measurements to generate an accurately georeferenced point cloud.

Classification Process:

Point cloud classification follows a semi-automated approach using TerraSolid software. First, algorithmic methods identify ground points using a progressive densification method. Next, vegetation points are classified based on height above ground and return characteristics. Buildings are identified using planar surface detection. Manual review and refinement ensures classification accuracy exceeds 98% when compared to control samples.

✨ Classification Accuracy ✨

Manual verification of randomly selected areas confirmed 98.5% classification accuracy, exceeding the project requirement of 95%. Ground classification achieved the highest accuracy at 99.2%, while vegetation classification ranged from 96.8-98.3% depending on density.

Client Case Study:

LiDAR data from this project was used by BC Hydro for corridor mapping of transmission lines. The high-resolution point cloud enabled precise measurement of vegetation encroachment and identification of potential hazard trees, leading to targeted maintenance operations and improved reliability.

Conclusion & Lessons Learned
✨ Key Learnings: ✨
  • Integration of hardware and software components is critical to efficient LiDAR workflows
  • Quality control at each stage ensures high-accuracy final products
  • Semi-automated classification with manual verification achieves optimal results
  • Careful flight planning significantly impacts data quality and project economics
Challenges & Solutions:
Challenge Solution
Weather delays Flexible scheduling and monitoring systems
Dense vegetation Multiple flight lines with increased overlap
Large data volumes Optimized storage and processing infrastructure
Building classification Enhanced algorithms with manual verification
